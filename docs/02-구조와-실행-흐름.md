
## 백엔드 구조

백엔드는 NestJS 기반이며, 레이어는 비교적 단순합니다.

- `controllers/`: HTTP 엔드포인트
- `services/`: 업무 로직
- `repositories/`: TypeORM 접근 로직
- `entities/`: DB 스키마
- `dto/`: 요청 검증
- `modules/`: 기능 모듈 묶음

실제 부팅 진입점:

- [main.ts](/Users/yeop/dev/wms/apps/server/src/main.ts)
- [app.module.ts](/Users/yeop/dev/wms/apps/server/src/app.module.ts)

## 프론트엔드 구조

프론트는 React + React Router + TanStack Query 구조입니다.

- `src/App.tsx`: 라우트 진입점
- `src/pages/*`: 주요 화면
- `src/components/*`: 공통 UI/기능 컴포넌트
- `src/hooks/queries/*`: 서버 상태 연동
- `src/lib/api.ts`: 실제 API 호출 정의

## 공통 타입 계층

[packages/types/index.ts](/Users/yeop/dev/wms/packages/types/index.ts)는 프론트와 백엔드 사이의 계약입니다.

여기를 보면 아래를 한 번에 알 수 있습니다.

## 실행 환경

현재 코드 기준 기본값:

- API prefix: `/api`
- DB host: `localhost`
- DB port: `3306`
- DB name: `wms`
- DB user/password: `root` / `root`
- API port: `3000`

필수에 가까운 환경 변수:

- `OPENAI_API_KEY`

- `PORT`

## 이 구조를 볼 때 주의할 점

- 이름이 풍부해 보여도 구현 범위는 그보다 좁을 수 있습니다.
- 특히 upload, packing, 3D 관련 기능은 "현재 서비스 코드가 실제로 무엇을 저장/반환하는지"까지 봐야 정확합니다.
- 따라서 화면 이름이나 타입 이름만 보고 기능을 추론하면 틀릴 가능성이 있습니다.
