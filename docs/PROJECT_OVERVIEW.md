# dnut VC - 프로젝트 개요

## 프로젝트 목적

물류 창고에서 출고 시 상품을 어떤 박스에 담을지 자동으로 계산해주는 dnut VC(Volume Calculator) 웹 애플리케이션.

엑셀로 관리되던 출고 데이터를 업로드하면 AI가 컬럼을 자동 매핑하고, 등록된 상품 치수와 박스 규격을 기반으로 최적의 박스 배정을 계산한다.

## 핵심 비즈니스 로직

### 1. 상품 마스터 관리

- 상품을 **상품 그룹(ProductGroup)** 단위로 묶어 관리
- 각 상품은 SKU, 이름, 가로/세로/높이(mm) 치수를 가짐
- 엑셀 업로드 시 AI가 SKU, 상품명, 치수 컬럼을 자동 추정 (OpenAI Structured Output)
- 치수는 `"300x200x100"` 같은 결합 포맷과 개별 컬럼 포맷 모두 지원

### 2. 박스 규격 관리

- 박스를 **박스 그룹(BoxGroup)** 단위로 묶어 관리
- 각 박스는 이름, 가로/세로/높이(mm), 가격(선택) 정보를 가짐
- 엑셀 일괄 업로드 지원
- 패킹 계산 시 박스 그룹을 선택하여 해당 그룹의 박스들로 배정

### 3. 출고 업로드 (AI 기반)

**업로드 플로우:**
1. 엑셀 파일 업로드 → 서버에서 파싱
2. 저장된 업로드 템플릿과 헤더 매칭 시도 (코사인 유사도)
3. 템플릿 매칭 실패 시 OpenAI로 컬럼 매핑 추론 (orderId, sku, quantity, recipientName, address)
4. 사용자가 매핑 결과 확인/수정
5. 확정 시 OutboundBatch + Orders + OutboundItems 생성
6. 매핑 결과를 템플릿으로 저장하여 재사용 가능

**복합 행(Compound Row) 처리:**
- 하나의 SKU 셀에 여러 상품이 줄바꿈으로 포함된 경우 (`(상품A / 1ea)\r\n(상품B / 2ea)`)
- `row-normalizer`가 이를 개별 행으로 분리

**상품 자동 매칭:**
- 출고 아이템의 SKU/상품명을 DB에 등록된 상품과 AI로 매칭
- OpenAI Structured Output + Zod 스키마로 매칭 결과 파싱
- 매칭 실패 시 `productId: null`로 저장 (데이터 유실 없음)

### 4. 패킹 계산

**그룹핑 옵션:**
- `ORDER` — 주문번호별로 그룹핑
- `RECIPIENT` — 수령인별로 그룹핑
- `ORDER_RECIPIENT` — 주문번호+수령인 조합으로 그룹핑

**패킹 알고리즘 (부피 기반 First-Fit Decreasing):**
1. 사용 가능한 박스를 부피 기준 오름차순 정렬
2. 그룹 내 모든 아이템의 총 부피 계산
3. 아이템이 물리적으로 들어가는지 확인 (3축 각각 비교)
4. 총 부피가 박스 유효 용량(90% 효율) 이내이면 해당 박스 배정
5. 적합한 박스가 없으면 "미지정" 박스로 배정
6. CBM(Cubic Meter)과 효율(사용부피/박스부피) 계산

**결과 저장:**
- `PackingResult` — 그룹별 요약 (박스명, 개수, 효율, CBM)
- `PackingResultDetail` — 주문-상품-박스 단위 상세 기록

### 5. 결과 내보내기

- 패킹 결과를 엑셀 파일로 다운로드
- 주문별 박스 배정, 상품 내역, CBM, 효율 포함

## 사용자 플로우

```
상품 그룹 생성 → 상품 엑셀 업로드 (AI 매핑)
                            ↓
박스 그룹 생성 → 박스 등록/업로드
                            ↓
출고 엑셀 업로드 (AI 매핑 + 상품 매칭)
                            ↓
패킹 계산 (그룹핑 옵션 + 박스 그룹 선택)
                            ↓
결과 확인 → 엑셀 내보내기
```
