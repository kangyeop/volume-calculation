# 구조와 아키텍처

## 디렉토리 구조

```text
volume-calculator/
├── apps/
│   ├── server/                  # NestJS 백엔드 API
│   │   ├── src/
│   │   │   ├── main.ts          # 진입점 (포트 3000)
│   │   │   ├── app.module.ts    # 루트 모듈
│   │   │   ├── controllers/     # HTTP 엔드포인트 (11개)
│   │   │   ├── services/        # 비즈니스 로직 (14개)
│   │   │   ├── repositories/    # TypeORM 데이터 접근
│   │   │   ├── entities/        # DB 스키마 (10개)
│   │   │   ├── dto/             # 요청 검증
│   │   │   └── modules/         # NestJS 모듈 + 패킹 알고리즘
│   │   └── uploads/             # 업로드 파일 임시 저장
│   │
│   └── web/                     # React 프론트엔드
│       └── src/
│           ├── App.tsx           # 라우트 정의
│           ├── pages/            # 페이지 컴포넌트
│           ├── components/       # 공통 UI 컴포넌트
│           ├── hooks/queries/    # React Query 서버 상태 관리
│           └── lib/api.ts        # API 클라이언트
│
├── packages/
│   └── types/index.ts           # 프론트/백엔드 공통 타입 계약
│
├── docs/                        # 프로젝트 문서
├── docker-compose.yml           # MySQL 컨테이너
├── turbo.json                   # Turbo 빌드 설정
└── pnpm-workspace.yaml          # pnpm 워크스페이스
```

| 레이어 | 기술 |
|--------|------|
| 패키지 매니저 | pnpm |

## 데이터베이스 스키마

### 엔티티 관계도

```text
ProductGroupEntity
    └── 1:N → ProductEntity

OutboundBatchEntity
    ├── 1:N → OrderEntity
    ├── 1:N → OutboundItemEntity
    ├── 1:N → PackingResultEntity
    └── 1:N → PackingResultDetailEntity

OrderEntity
    └── 1:N → OutboundItemEntity (cascade delete)

OutboundItemEntity
    ├── N:1 → OutboundBatchEntity
    ├── N:1 → OrderEntity
    └── N:1 → ProductEntity (nullable)

ProjectEntity
    └── 1:N → OutboundEntity (레거시)

BoxEntity (전역, FK 없음)
```

### 엔티티 상세

| 엔티티 | 역할 | 주요 필드 |
|--------|------|----------|
| ProductGroupEntity | 상품 그룹 | id, name |
| ProductEntity | 상품 마스터 | id, sku, name, width, length, height, productGroupId |
| BoxEntity | 박스 규격 (전역) | id, name, width, length, height, price? |
| OutboundBatchEntity | 출고 배치 | id, name |
| OrderEntity | 주문 | id, orderId, recipientName, address, status, outboundBatchId |
| OutboundItemEntity | 출고 아이템 | id, orderId, sku, quantity, outboundBatchId, productId? |
| PackingResultEntity | 패킹 결과 요약 | id, boxName, packedCount, efficiency, totalCBM, groupLabel, outboundBatchId |
| PackingResultDetailEntity | 패킹 상세 기록 | id, orderId, sku, productName, quantity, boxName, boxNumber, boxCBM, efficiency |
| ProjectEntity | 프로젝트 (레거시) | id, name |
| OutboundEntity | 출고 (레거시, 프로젝트 스코프) | id, orderId, sku, quantity, projectId, productId? |

### 주의사항

- `OutboundEntity`와 `ProjectEntity`는 레거시 모델로, 현재 UI는 `OutboundBatchEntity`/`OutboundItemEntity` 기반으로 동작
- `OutboundItemEntity.productId`는 nullable: 상품 매칭에 실패한 아이템도 저장됨
- `OrderEntity.status`는 `PENDING | PROCESSING | COMPLETED` 3가지 상태
