# 데이터 흐름

이 문서는 주요 비즈니스 흐름에서 데이터가 어떻게 이동하는지를 설명합니다.

## 1. 상품 업로드 흐름

```text
[사용자: 엑셀 파일 선택]
        │
        ▼
POST /product-upload/parse?groupId=xxx (FormData: file)
        │
        ▼
┌─ ProductUploadController.parse() ─────────────────────┐
│  ExcelService.parseExcelFile(file)                     │
│    → { headers[], rows[], rowCount }                   │
│                                                        │
│  AIService.mapProductColumns(headers, sampleRows)      │
│    → OpenAI에 헤더+샘플 데이터 전송                       │
│    → { mapping: { sku, name, dimensions }, format }    │
│                                                        │
│  응답: { sessionId, headers, rows, mapping, fileName } │
└────────────────────────────────────────────────────────┘
        │
        ▼
[사용자: 매핑 결과 확인 → 확인 클릭]
        │
        ▼
POST /product-upload/confirm { groupId, rows, mapping }
        │
        ▼
┌─ ProductUploadController.confirm() ───────────────────┐
│  DataTransformerService.transformProductRows()          │
│    → rows + mapping으로 Product 배열 생성               │
│    → 치수 파싱: "100x200x300" → { w:100, l:200, h:300}│
│    → 또는 separate 컬럼에서 개별 추출                    │
│                                                        │
│  ProductsService.createBulk(groupId, products)         │
│    → DB에 일괄 저장                                     │
│                                                        │
│  응답: { imported: 42 }                                │
└────────────────────────────────────────────────────────┘
```

### 관련 파일
- `controllers/productUpload.controller.ts`
- `services/productUpload.service.ts`
- `services/ai.service.ts` (mapProductColumns)
- `services/dataTransformer.service.ts` (transformProductRows)
- `services/excel.service.ts` (parseExcelFile)

## 2. 출고 업로드 흐름

```text
[사용자: 출고 엑셀 파일 선택]
        │
        ▼
POST /upload/outbound-direct (FormData: file)
        │
        ▼
┌─ UploadController.uploadOutboundDirect() ─────────────┐
│                                                        │
│  ① 엑셀 파싱                                           │
│  ExcelService.parseExcelFile(file)                     │
│    → { headers[], rows[] }                             │
│                                                        │
│  ② AI 컬럼 매핑                                        │
│  AIService.mapOutboundColumns(headers, sampleRows)     │
│    → OpenAI에 헤더+샘플 전송                             │
│    → { mapping: { orderId, sku, quantity,               │
│                   recipientName, address } }            │
│                                                        │
│  ③ 데이터 변환                                          │
│  DataTransformerService.transformAndMapOutbound()       │
│    → columnMapping 기반으로 rows → 주문/아이템 구조 변환  │
│    → orderId 기준으로 그룹핑하여 Order 생성              │
│                                                        │
│  ④ 상품 매칭                                            │
│  ProductsService.findAllForMatching()                  │
│    → 전체 상품 조회 → byName, bySku 두 개의 Map 생성     │
│                                                        │
│  매칭 우선순위:                                          │
│    1순위: 상품명(name)으로 매칭                           │
│    2순위: SKU로 매칭                                     │
│    매칭 실패 시: productId = null로 저장                  │
│                                                        │
│  ⑤ 배치 생성 및 저장                                     │
│  OutboundBatchService.create(batchName)                 │
│    → "YYYY-MM-DD_outbound_NNN" 형식 자동 생성            │
│                                                        │
│  UploadRepository.createOutboundsWithOrder()            │
│    → Order + OutboundItem 벌크 INSERT                   │
│                                                        │
│  응답: { imported, unmatched[], batchName, batchId,     │
│          totalRows }                                    │
└────────────────────────────────────────────────────────┘
```

### 상품 매칭 상세

```text
엑셀 행의 SKU 값 (예: "나나시 클리어파일")
        │
        ├─ productByName["나나시 클리어파일"] → 있으면 매칭 성공
        │
        └─ productBySku["나나시 클리어파일"] → 있으면 매칭 성공
                │
                └─ 둘 다 없으면 → unmatched에 추가, productId = null
```

### 관련 파일
- `controllers/upload.controller.ts`
- `services/upload.service.ts` (uploadAndSaveDirect)
- `services/ai.service.ts` (mapOutboundColumns)
- `services/dataTransformer.service.ts` (transformAndMapOutbound)
- `repositories/upload.repository.ts` (createOutboundsWithOrder)

## 3. 패킹 계산 흐름

```text
[사용자: 그룹핑 옵션 선택 → 계산 클릭]
        │
        ▼
POST /outbound-batches/:batchId/packing/calculate
     { groupingOption: "ORDER" | "RECIPIENT" | "ORDER_RECIPIENT" }
        │
        ▼
┌─ PackingController.calculate() ───────────────────────┐
│                                                        │
│  ① 데이터 조회                                          │
│  OutboundService.findAll(batchId) → 출고 아이템 목록     │
│  ProductsService.findAll() → 전체 상품 (치수 포함)       │
│  BoxesService.findAll() → 전체 박스 규격                 │
│                                                        │
│  ② 그룹핑                                               │
│  PackingService.groupOutbounds(outbounds, option)      │
│    ORDER → orderId별 그룹                               │
│    RECIPIENT → recipientName별 그룹                     │
│    ORDER_RECIPIENT → orderId_recipientName별 그룹       │
│                                                        │
│  ③ 각 그룹별 패킹 계산                                   │
│  for (group of groups) {                               │
│    outboundItems → SKU[] (치수+수량 배열)로 변환          │
│    calculatePacking(skus, boxes)                        │
│      → 패킹 알고리즘 실행 (05-패킹-알고리즘.md 참조)      │
│      → { boxes[], unpackedItems[], totalCBM, efficiency}│
│  }                                                     │
│                                                        │
│  ④ 결과 저장                                             │
│  기존 결과 삭제 후 새로 생성                               │
│  PackingResult (그룹별 요약) 저장                         │
│  PackingResultDetail (아이템별 상세) 저장                  │
│                                                        │
│  응답: PackingRecommendation                            │
│  { groups[], totalCBM, totalEfficiency }                │
└────────────────────────────────────────────────────────┘
```

### 관련 파일
- `controllers/packing.controller.ts`
- `services/packing.service.ts`
- `modules/packing/algorithms/packing.algorithm.ts`
- `repositories/packing-results.repository.ts`
- `repositories/packing-result-details.repository.ts`

## 4. 패킹 결과 내보내기 흐름

```text
[사용자: 엑셀 내보내기 클릭]
        │
        ▼
GET /outbound-batches/:batchId/packing/export
        │
        ▼
┌─ PackingController.export() ──────────────────────────┐
│  PackingResultDetailRepository.find(batchId)           │
│    → 저장된 상세 기록 전체 조회                          │
│                                                        │
│  ExcelService.exportPackingResults(details)             │
│    → 엑셀 파일 생성                                     │
│    → 컬럼: orderId, recipientName, sku, productName,   │
│           quantity, boxName, boxNumber, boxCBM,         │
│           efficiency, unpacked, unpackedReason          │
│                                                        │
│  응답: Excel 파일 (Blob)                                │
└────────────────────────────────────────────────────────┘
```
